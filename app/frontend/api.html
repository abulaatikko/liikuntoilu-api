API documentation (v1)

Last updated: 2015-11-14

General

  • Only GET methods supported.
  • {..} in the URLs are placeholders for the dynamic data. Replace them by the real values when necessary.
  • {id} can be 0 (bad practice)
  • Response cache is used. Observe Cache-Control HTTP header.

Methods

GET /api/v1/participants

The method returns all the participants.

Field Type Description
id integer Identifier of the participant.
name string Name of the participant.
active boolean The field tells if the participant is active.

Example: http://liikuntoilu.net/api/v1/participants

GET /api/v1/participants/{id}

The method returns one participant.

Field Type Description
id integer Identifier of the participant.
name string Name of the participant.
active boolean The field tells if the participant is active.

Example: http://liikuntoilu.net/api/v1/participants/0
